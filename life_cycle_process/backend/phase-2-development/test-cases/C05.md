## TC-C05-BLD-001: 编译检查

- **Given**: `backend/src/response_handler.py` 存在
- **When**: `python -m py_compile backend/src/response_handler.py`
- **Then**: 编译成功

## TC-C05-LOG-001: 逻辑 — handle_non_stream 返回 JSONResponse

- **Given**: target_response status=200, body 为 Anthropic Message JSON
- **When**: `handle_non_stream(target_response, "anthropic")`
- **Then**: 返回 JSONResponse，status=200，body 不含 image block

## TC-C05-LOG-002: 逻辑 — handle_stream 返回 StreamingResponse

- **Given**: target_stream yield 3 个 SSE chunk
- **When**: `handle_stream(target_stream, "anthropic")`
- **Then**: 返回 StreamingResponse，media_type="text/event-stream"

## TC-C05-LOG-003: 逻辑 — StreamingResponse 正确转发 SSE 事件

- **Given**: target_stream = ["data: chunk1", "data: chunk2", "data: [DONE]"]
- **When**: handle_stream
- **Then**: 客户端收到的内容与 target_stream 一致

## TC-C05-LOG-004: 逻辑 — Anthropic 格式下 SSE 事件类型正确

- **Given**: target_stream 含 content_block_delta / message_delta 等
- **When**: 流式转发
- **Then**: SSE 事件类型保持 Anthropic 格式（message_start / content_block_start / content_block_delta / content_block_stop / message_delta / message_stop）

## TC-C05-LOG-005: 逻辑 — 非流式请求的 Content-Type 正确

- **Given**: 非流式
- **When**: handle_non_stream
- **Then**: response header `Content-Type: application/json`

## TC-C05-LOG-006: 逻辑 — 流式请求的 Content-Type 正确

- **Given**: 流式
- **When**: handle_stream
- **Then**: response header `Content-Type: text/event-stream`

## TC-C05-SYS-001: 系统行为 — §4: RESPONDING 非流式 200

**来源：系统交互规格 §4 — RESPONDING**

- **Given**：target response 200 + 正常 body
- **When**：handle_non_stream
- **Then**：返回 200，body 完整
- **PASS 判定**：status 200 + body 可解析
- **FAIL 判定**：返回空或截断

## TC-C05-SYS-002: 系统行为 — §4: RESPONDING 流式每个 chunk

**来源：系统交互规格 §4 — RESPONDING**

- **Given**：target stream 逐 token 返回
- **When**：handle_stream
- **Then**：客户端逐 token 收到 SSE 事件
- **PASS 判定**：至少收到 1 个 content_block_delta + 1 个 message_stop
- **FAIL 判定**：只收到 message_stop 无 delta

## TC-C05-SYS-003: 系统行为 — §7.3: 识图进行中不发送事件

**来源：系统交互规格 §7.3 — 加载态 — 识图进行中**

- **Given**：管道正在识图（阶段 3）
- **When**：检查是否有 SSE 事件发出
- **Then**：无任何 SSE 事件，连接保持
- **PASS 判定**：identify 阶段无 SSE 输出，连接不断开
- **FAIL 判定**：识图期间发送了事件

## TC-C05-SYS-004: 系统行为 — §7.3: 目标模型推理中逐 token 转发

**来源：系统交互规格 §7.3 — 加载态 — 目标模型推理中**

- **Given**：target stream 逐 token 产生
- **When**：handle_stream
- **Then**：客户端实时收到每个 token
- **PASS 判定**：客户端收到增量 text_delta
- **FAIL 判定**：所有 token 一起到达（非流式）
