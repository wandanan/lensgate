## TC-C04-BLD-001: 编译检查

- **Given**: `backend/src/target_client.py` 存在
- **When**: `python -m py_compile backend/src/target_client.py`
- **Then**: 编译成功

## TC-C04-LOG-001: 逻辑 — forward 发送正确 endpoint

- **Given**: mock httpx, TargetModelConfig(api_base="https://ark.cn-beijing.volces.com/api/coding")
- **When**: `forward(request_body, config)`
- **Then**: POST URL 为 `https://ark.cn-beijing.volces.com/api/coding/v1/messages`

## TC-C04-LOG-002: 逻辑 — forward 携带正确 auth header

- **Given**: config.api_key = "test-key-123"
- **When**: forward
- **Then**: 请求 header 含 `x-api-key: test-key-123` 和 `anthropic-version: 2023-06-01`

## TC-C04-LOG-003: 逻辑 — forward 非流式返回 JSON

- **Given**: mock Target API 返回 200 + Anthropic Message JSON
- **When**: `forward(request_body, config)`
- **Then**: 返回 httpx.Response，status=200，body 可 JSON 解析

## TC-C04-LOG-004: 逻辑 — forward_stream 逐行 yield SSE

- **Given**: mock Target API 返回 SSE stream（3 行 data）
- **When**: `forward_stream(request_body, config)`
- **Then**: async generator yield 3 行，每行以 "data: " 开头

## TC-C04-SYS-001: 系统行为 — §4: FORWARDING 目标模型 200

**来源：系统交互规格 §4 — FORWARDING → RESPONDING**

- **Given**：mock Target API 返回 200
- **When**：forward
- **Then**：response.status_code == 200
- **PASS 判定**：status 200
- **FAIL 判定**：非 200 或异常

## TC-C04-SYS-002: 系统行为 — §4: FORWARDING 目标模型 5xx → 503

**来源：系统交互规格 §4 — FORWARDING → 503**

- **Given**：mock Target API 返回 500
- **When**：forward
- **Then**：返回 503，body 含 "target_model_unavailable"
- **PASS 判定**：status 503 + error 消息
- **FAIL 判定**：透传 500

## TC-C04-SYS-003: 系统行为 — §4: FORWARDING 超时 → 504

**来源：系统交互规格 §4 — FORWARDING → 504**

- **Given**：mock Target API 触发 TimeoutException
- **When**：forward
- **Then**：返回 504，body 含 "target_model_timeout"
- **PASS 判定**：status 504
- **FAIL 判定**：其他错误码

## TC-C04-SYS-004: 系统行为 — request_body.stream=true 时使用 stream 路径

- **Given**：request_body 含 "stream": true
- **When**：管道到 forwarding 阶段
- **Then**：调用 forward_stream 而非 forward
- **PASS 判定**：forward_stream 被调用
- **FAIL 判定**：调用了 forward（非流式）

## TC-C04-SYS-005: 系统行为 — §7.1: 客户端断连取消上游

**来源：系统交互规格 §7.1 — 客户端断连**

- **Given**：forward_stream 进行中，客户端已断开
- **When**：尝试 yield 下一个 SSE chunk
- **Then**：httpx stream 被关闭，资源释放
- **PASS 判定**：stream 正确关闭，无连接泄漏
- **FAIL 判定**：未关闭或抛未捕获异常
