## TC-C08-LOG-001: 逻辑 — structlog 输出 JSON 格式

- **Given**: 调用 setup_logging()
- **When**: logger.info("test", key="value")
- **Then**: stdout 输出可被 json.loads 解析，含 event/level/timestamp
- **PASS 判定**: json.loads(output_line) 成功 + 含 event 键
- **FAIL 判定**: 非 JSON

## TC-C08-LOG-002: 逻辑 — 请求日志含 method/path/status/duration

- **Given**: 中间件已注册
- **When**: 处理 HTTP 请求
- **Then**: 日志含 method/path/status_code/duration_ms
- **PASS 判定**: 4 个字段均存在且值正确
- **FAIL 判定**: 缺字段

## TC-C08-LOG-003: 逻辑 — 图片处理日志含 image_count/vision_duration

- **Given**: 处理含图请求
- **When**: Vision 阶段完成
- **Then**: 日志含 image_count (>= 1) 和 vision_duration_ms (> 0)
- **PASS 判定**: image_count >= 1，vision_duration_ms > 0
- **FAIL 判定**: 字段缺失

## TC-C08-LOG-004: 逻辑 — 错误日志含 traceback

- **Given**: 发生异常
- **When**: exception handler 捕获
- **Then**: 日志含 exception 或 traceback 字段
- **PASS 判定**: exception/traceback 非空
- **FAIL 判定**: 仅错误消息无堆栈

## TC-C08-API-001: API — GET /health 返回 status ok

- **Given**: 服务运行中
- **When**: GET /health
- **Then**: 200，body 含 status/version/timestamp
- **PASS 判定**: status 200 + 3 个字段均存在
- **FAIL 判定**: 缺字段

## TC-C08-API-002: API — 请求产生 JSON 日志行

- **Given**: 服务运行中
- **When**: 发任意合法请求
- **Then**: stdout 有新 JSON 行含 method 和 path
- **PASS 判定**: 新 JSON 行含 method/path
- **FAIL 判定**: 无输出
