## format_detector.py 设计要点

### detect_format(path, body=None)
- 基于路径检测格式：`/v1/messages` → `"anthropic"`，`/v1/chat/completions` → `"openai"`
- 自动剥离 query string (?) 和尾部斜杠
- 未知路径抛出 ValueError

### parse_anthropic_request(body) → ProxyRequest
- 解析 `model`, `max_tokens`(默认 4096), `stream`(默认 False), `system`
- `messages[].content`：
  - string → `[TextBlock(text=...)]`
  - list → 逐项按 type 分发：
    - `type="text"` → TextBlock
    - `type="image"` + `source.type="base64"` → ImageBlock（解码 base64，source_type="base64"）
    - `type="image"` + `source.type="url"` → ImageBlock（image_data=b""，source_type="url"，留待 Image Extractor 下载）
    - `type="tool_use"` / `type="tool_result"` → TextBlock（JSON 序列化，透明透传）
    - 其他/未知 type → TextBlock
- `original_body` 完整保留原始请求体

### parse_openai_request(body) → ProxyRequest
- 解析 `model`, `max_tokens`, `stream`（OpenAI 无 system 顶层字段）
- `messages[].content`：
  - string → `[TextBlock]`
  - list → 逐项按 type 分发：
    - `type="text"` → TextBlock
    - `type="image_url"` + data URI (`data:image/...;base64,...`) → ImageBlock（解码 base64，source_type="data_uri"）
    - `type="image_url"` + 普通 URL → ImageBlock（image_data=b""，source_type="url"）
    - 其他 → TextBlock

### ImageBlock 位置信息
- 每个 ImageBlock 携带 `message_index`（消息在 messages 中的下标）和 `block_index`（block 在 content 数组中的下标），供后续 Image Extractor 定位。

## 路由处理器

| 端点 | 处理器 | 行为 |
|------|--------|------|
| `POST /v1/messages` | `handle_anthropic_messages` | 解析 JSON → parse_anthropic_request → 返回 parsed 确认 |
| `POST /v1/chat/completions` | `handle_openai_chat_completions` | 解析 JSON → parse_openai_request → 返回 parsed 确认 |

**错误处理**：
- JSON 解析失败 → HTTP 400 `{"error": "invalid_request"}`
- 解析异常 → HTTP 400 `{"error": "parse_error: ..."}`

## 注意事项

1. **URL 图片延迟下载**：Anthropic `source.type="url"` 和 OpenAI 普通 URL 图片的 `image_data` 在解析阶段为 `b""`，实际下载由后续 Image Extractor 任务完成。
2. **tool_use/tool_result 透传**：序列化为 JSON 字符串存入 TextBlock，后续 Request Rewriter 可从 `original_body` 完整恢复原始 block。
3. **detect_format 兼容 query string**：路径匹配前先剥离 `?` 及后续参数。
4. **路由为阶段实现**：当前返回 parsed 确认 JSON，完整代理管线（识图→重写→转发→响应）将在后续任务中接入。
5. **无 .env 依赖**：format_detector 和 routes 不依赖配置，可独立测试运行。
