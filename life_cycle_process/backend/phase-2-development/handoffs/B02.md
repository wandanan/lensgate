## image_extractor.py 设计要点

### 三个公开函数

| 函数 | 签名 | 职责 |
|------|------|------|
| `has_images` | `(request: ProxyRequest) -> bool` | IMAGE_CHECK 阶段：扫描 messages 中是否有 ImageBlock |
| `extract_images` | `async (request: ProxyRequest) -> list[ImageBlock]` | EXTRACTING 阶段：解析 image_data（base64/URL/data_uri） |
| `validate_image_format` | `(block: ImageBlock) -> bool` | 校验 media_type 是否在白名单中 |

### 数据来源处理

| 来源类型 | 处理方式 | 实现位置 |
|----------|---------|---------|
| Anthropic `source.type="base64"` | base64.b64decode(source_data) | `_resolve_image_data` |
| Anthropic `source.type="url"` | httpx.AsyncClient.get() 下载 | `_download_image` |
| OpenAI data URI | 正则解析 `data:image/<type>;base64,<data>` + base64 decode | `_resolve_image_data` |
| OpenAI HTTPS URL | httpx.AsyncClient.get() 下载 | `_download_image` |

### 图片格式白名单

`image/png`, `image/jpeg`, `image/webp`, `image/gif`（对齐 Qwen 3.7 Plus 能力）

### 限界上下文

- 不调用视觉模型（由 Vision Client / C02 负责）
- 不做请求体重写（由 Request Rewriter / C03 负责）
- 仅做检测 + 二进制数据提取

## models.py 变更

`ImageBlock` 新增两个字段：

- `source_data: str = ""` — 承载原始 base64 字符串、URL 或 data URI（由 parser 设置，extractor 消费）
- `is_error: bool = False` — 当 image_data 解析失败时标记为 True（URL 下载失败/无效 data URI/未知 source_type）

所有现有字段改为有默认值以兼容新字段，**现有 6 个 model 测试和其余测试全部通过**，无破坏性变更。

## 注意事项

1. **extract_images 是 async 函数**：URL 下载需要异步 HTTP。调用方必须使用 `await`。
2. **is_error 不抛异常**：URL 下载失败、无效 data URI、未知 source_type 等情况均标记 `is_error=True` 并继续，由下游 pipeline（C03 Request Rewriter）注入降级文本 `"[图片下载失败]"`。
3. **message_index/block_index 由 extractor 设置**：基于实际遍历位置设置，覆盖 parser 可能设置的值。用于下游 C03 回填识别结果到正确位置。
4. **下载超时 30s**：与 vision_timeout 默认值一致，可在后续版本中提取为配置。
5. **httpx.AsyncClient 按块创建**：每个 URL 下载创建新的 client context，确保连接正确关闭。对于单图场景完全够用，多图并行优化留给 C03 或后续任务。
6. **source_data 字段由 parser (B01) 填充**：B01 的 format_detector 在创建 ImageBlock 时需要将原始 base64 字符串/URL/data URI 写入 source_data 字段。
