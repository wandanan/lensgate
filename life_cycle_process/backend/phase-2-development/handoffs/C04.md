## target_client.py 设计要点

- **TargetModelClient(router: ModelRouter)** — 构造函数接收 ModelRouter 实例（为后续管道阶段预留）
- **forward(request_body, config) -> httpx.Response** — 非流式转发，60s 超时
- **forward_stream(request_body, config) -> AsyncGenerator[str, None]** — SSE 流式转发，120s 超时，逐行 yield
- **_build_headers(config)** — 静态方法，构造 `x-api-key` / `anthropic-version: 2023-06-01` / `Content-Type: application/json`
- **close()** — 关闭底层 httpx.AsyncClient（支持多次调用，幂等）

### API 端点

- URL: `{api_base}/v1/messages`（Anthropic Messages API 兼容端点）
- 默认 api_base: `https://ark.cn-beijing.volces.com/api/coding`

### 错误处理策略

| 异常 | 行为 | 说明 |
|------|------|------|
| `httpx.HTTPStatusError` | re-raise | 由 ResponseHandler/C07 映射为 503 |
| `httpx.TimeoutException` | re-raise | 由 ResponseHandler/C07 映射为 504 |
| 目标 API 返回 500 | 透传 response（status=500） | 不做状态码转换，C07 负责 |

**设计理由**: TargetClient 只负责原始 HTTP 转发，错误码转换是 C07 ResponseHandler 的职责，符合单一职责和分层原则。

### Client 生命周期

- httpx.AsyncClient 采用懒初始化（`_get_client`）
- `close()` 方法显式释放连接，支持多次调用（幂等）
- 管道集成时，建议在 shutdown 阶段调用 `close()`

## 注意事项

1. **错误码不在此层转换**: 系统交互规格定义的 503/504 映射由 C07 ResponseHandler 负责。TargetClient 仅透传原始响应或 re-raise 异常。C07 需要 catch `httpx.HTTPStatusError` → 503, `httpx.TimeoutException` → 504。
2. **stream: true 强制注入**: `forward_stream` 始终在发送给目标模型的请求体中注入 `stream: true`，无论传入的 request_body 是否已包含该字段。这确保上游 SSE 流被正确触发。
3. **空行过滤**: `forward_stream` 跳过上游返回的空行（SSE 协议中使用空行分隔事件，但 aiter_lines 已处理换行，额外空行被过滤以提高健壮性）。
4. **httpx.AsyncClient 共享**: 使用单例 AsyncClient（懒初始化），利用连接池复用 TCP 连接，减少握手开销。调用方应在 shutdown 时调用 `close()`。
5. **api_base 末尾无斜杠**: endpoint 构建方式为 `f"{api_base}/v1/messages"`，配置 api_base 时末尾不应带斜杠。当前默认值 `https://ark.cn-beijing.volces.com/api/coding` 符合此约定。
6. **C07 集成点**: C07 的 `handle_response()` 需要：
   - 接收 `forward()` 返回的 `httpx.Response`
   - 检查 `response.status_code`，5xx → 返回 503 + `target_model_unavailable`
   - 包装 `try/except httpx.TimeoutException` → 返回 504 + `target_model_timeout`
   - 成功时从 response body 提取内容返回客户端
