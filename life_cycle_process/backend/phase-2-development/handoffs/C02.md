## vision_client.py 设计要点

### QwenVisionClient 类

| 方法 | 签名 | 职责 |
|------|------|------|
| `__init__` | `(config: ProxyConfig)` | 从 ProxyConfig 读取 vision_api_key / vision_base_url / vision_model / vision_timeout |
| `recognize` | `async (image: ImageBlock) -> str` | 识别单张图片，失败返回降级文本 |
| `recognize_batch` | `async (images: list[ImageBlock]) -> list[str]` | 并行识别多张图片，单失败不阻塞 |

### API 调用细节

- **Endpoint**: `POST {vision_base_url}/v1/chat/completions`（vision_base_url 默认 `https://coding.dashscope.aliyuncs.com`）
- **Auth**: `Authorization: Bearer {vision_api_key}`
- **Model**: `qwen3.7-plus`
- **max_tokens**: 2000
- **Content**: `image_url` block (data URI) + `text` block (recognition prompt)

### 降级策略

| 场景 | 行为 |
|------|------|
| API 返回非 200 | 返回 `"[图片无法识别]"` |
| 超时 (config.vision_timeout) | 返回 `"[图片无法识别]"` |
| JSON 解析失败 | 返回 `"[图片无法识别]"` |
| 网络错误 (RequestError) | 返回 `"[图片无法识别]"` |
| 429 限流 | 等待 1s 重试 1 次，仍失败则降级 |
| image_data 为 None | 立即返回 `"[图片无法识别]"`（不发 API 请求） |

### 并行策略

`recognize_batch` 使用 `asyncio.gather(*tasks, return_exceptions=True)`：
- 多张图片并发调用，总耗时 ≈ 最慢的单张耗时
- `return_exceptions=True` 确保单个失败不阻塞其余图片
- 任何被 gather 捕获的 Exception 被转换为 `"[图片无法识别]"`

### 限界上下文

- 不处理请求解析（由 Format Detector / B01 负责）
- 不下载 URL 图片（由 Image Extractor / B02 `extract_images` 完成，vision_client 只消费 `image_data` 字段）
- 不重写请求体（由 Request Rewriter / C03 负责）
- 仅负责图片 → 文字描述的转换

## 注意事项

1. **image_data 为 None 的早期返回**：ImageBlock 的 is_error=True 时 image_data 为 None，recognize 直接返回降级文本，不会发送 API 请求。
2. **media_type 回退**：当 ImageBlock.media_type 为空字符串时，recognize 使用 `"image/png"` 作为默认值（兼容格式检测失败场景）。
3. **httpx.AsyncClient 每次调用创建**：每个 recognize 调用创建独立的 client context，确保连接正确关闭。对于单图场景完全够用。
4. **Coding Plan URL 对齐**：vision_base_url 使用 `https://coding.dashscope.aliyuncs.com`（Coding Plan 专属），与 api-integration-reference.md 一致。
5. **降级文本不抛异常**：所有错误场景均返回 `"[图片无法识别]"`，确保 pipeline 不会因识图失败而中断（对齐系统交互规格 §4 RECOGNIZING 降级路径）。
6. **429 重试使用 asyncio.sleep(1.0)**：测试中 mock 了 sleep，实际运行时会真实等待 1s。
