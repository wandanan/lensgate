## response_handler.py 设计要点

- **ResponseHandler** — 无状态类，将 TargetClient 的原始输出转换为 FastAPI 响应对象
- **handle_non_stream(target_response, source_format) -> JSONResponse** — 读取 httpx.Response body，返回 JSONResponse
- **handle_stream(target_stream_generator, source_format) -> StreamingResponse** — 包装 async generator，返回 text/event-stream
- **_sse_generator(source_generator) -> AsyncGenerator[str, None]** — 逐行 yield，SSE 事件保持原样

### 方法签名

| 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|
| `handle_non_stream` | `target_response: httpx.Response`, `source_format: str` | `JSONResponse` | 非流式响应，Content-Type: application/json |
| `handle_stream` | `target_stream_generator: AsyncGenerator[str, None]`, `source_format: str` | `StreamingResponse` | 流式响应，media_type: text/event-stream |
| `_sse_generator` | `source_generator: AsyncGenerator[str, None]` | `AsyncGenerator[str, None]` | 内部 SSE 行转发器 |

### SSE 事件类型

Anthropic 格式 SSE 事件类型全部透传，不做修改：
- `message_start` — 消息开始
- `content_block_start` — 内容块开始
- `content_block_delta` — 内容增量（逐 token）
- `content_block_stop` — 内容块结束
- `message_delta` — 消息增量（含 stop_reason）
- `message_stop` — 消息结束

### 设计决策

1. **格式无关**：Anthropic 和 OpenAI 格式均返回 JSONResponse/StreeamingResponse，不做格式转换
2. **状态码透传**：target_response.status_code 直接传递给 JSONResponse，错误码映射由 C07 负责
3. **SSE 行透传**：TargetClient.forward_stream() 返回的每行以 `data: ` 开头，ResponseHandler 不做任何修改
4. **无状态**：ResponseHandler 可跨请求复用，多个并发调用之间无共享状态
5. **空生成器支持**：如果 source_generator 不 yield 任何行（如识图阶段），StreamingResponse 返回空流，连接保持

## 注意事项

1. **C07 错误映射集成点**：ResponseHandler 不做错误码转换，C07 需要：
   - 调用 `handle_non_stream` 前，catch `httpx.HTTPStatusError` → 503 + `target_model_unavailable`
   - 调用前 catch `httpx.TimeoutException` → 504 + `target_model_timeout`
   - ResponseHandler 本身只是将 httpx.Response 转为 FastAPI Response
2. **source_format 参数预留**：当前两种格式处理逻辑相同，参数仅供未来格式特定处理使用
3. **SSE 行格式约定**：ResponseHandler 假设上游传入的每行已经是完整 SSE 行（以 `data: ` 开头），不做补全或格式化
4. **空行已在上游过滤**：TargetClient.forward_stream() 已过滤空行，ResponseHandler 不做二次过滤
5. **流式连接生命周期**：StreamingResponse 在客户端断连时自动终止 async generator，无需手动管理
