## model_router.py 设计要点

- **ModelRouter.__init__(configs: dict[str, TargetModelConfig])** — 接收路由表，key 为配置名称
- **resolve(requested_model: str) -> TargetModelConfig** — 大小写不敏感 substring 匹配，无匹配时 fallback 到 `configs["default"]`
- **build_router(config: ProxyConfig) -> ModelRouter** — 从 ProxyConfig 构建 default + glm 两个 TargetModelConfig

### resolve 匹配逻辑

1. 将 `requested_model` 转小写
2. 遍历 configs 的 key，小写后检查是否为 `target` 的子串
3. 首个命中即返回对应 TargetModelConfig
4. 全部未命中则返回 `configs["default"]`

### 边界情况处理

- 空字符串 `""` 作为 model 名 → 所有非空 key 不匹配 → fallback 到 default
- key 为 `"default"` 时，`resolve("default-model")` → `"default" in "default-model"` → 返回 default config

## 注意事项

1. **ModelRouter 不执行运行时校验**：配置的 `api_base` / `api_key` 可能为空字符串，由调用方（后续管道阶段）在发出 HTTP 请求时才报错
2. **configs dict 的遍历顺序**：Python 3.7+ dict 保持插入序，"default" 输入顺序靠后。但 `resolve` 在遍历时会 checking all keys，顺序不影响 fallback 的正确性；如有多个 key 均为 substring，首次命中即返回（e.g. `"glm"` 和 `"glm-5"` 两个 key 时 `resolve("glm-5.1")` 返回先遍历到的配置 — 当前仅 2 个 key 不存在此冲突）
3. **build_router 依赖 ProxyConfig**：glm 配置字段（model/base_url/api_key）只要有一个为空，对应的 TargetModelConfig 已有但可能不可用。后续任务（C02 或管道集成阶段）需处理 fallback 到 default 的逻辑
4. **app.py 的 timestamp** 使用 `datetime.now(timezone.utc)`，与 Python 3.11+ 推荐的 UTC 感知写法一致（旧版 `datetime.utcnow()` 在 3.12 已 deprecated）
